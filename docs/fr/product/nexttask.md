# Tâches récurrentes

Dans dodone, les tâches récurrentes peuvent différer légèrement d'autres outils. Plus précisément, il s'agit de « la tâche suivante ». Les tâches récurrentes futures ne sont pas créées à l'avance ; à la place, on vous invite à créer la tâche suivante lorsque vous terminez la tâche actuelle. Par exemple, si vous définissez une tâche récurrente quotidienne, lorsque vous la terminez, on vous proposera de créer la tâche suivante et de définir l'heure pour le lendemain.

Si vous définissez la récurrence sur une tâche parente, les sous-tâches seront automatiquement copiées lors de la création de la tâche suivante.

Les tâches parentes et enfant peuvent toutes deux avoir des sous-tâches, et toutes suivent les règles décrites ci-dessus.

## Propriétés des tâches récurrentes

- a : Répéter tous les N jours
- b : Intervalle de repos par cycle. Si défini sur 2, le premier cycle se repose 1 jour, le deuxième ne se repose pas ; si défini sur 3, le premier cycle se repose 1 jour, le deuxième se repose 2 jours, le troisième ne se repose pas.
- endDate : Date de fin. Après cette date, il n'y aura plus d'invite pour créer la tâche suivante.
